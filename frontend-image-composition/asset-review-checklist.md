# Asset Review Checklist

## Review Questions

1. Does the asset have clean edges and usable transparency?
2. Does it avoid the generic AI-generated glow or template look?
3. If it depicts a character, are anatomy, hands, face, and proportions stable?
4. If it is an icon, is the meaning immediately readable?
5. Does it match the page’s typography, density, and visual language?
6. Does it improve hierarchy instead of distracting from content?
7. Would this asset feel acceptable in a production UI rather than only in a demo?

## Classification

### Use Directly

- Clean edges
- Fits the page language
- No obvious AI artifacts
- Improves the target area

### Temporary, Replace Later

- Mostly usable
- Some AI artifacts or over-stylization remain
- Acceptable as a placeholder but not ideal for long-term use

### Do Not Integrate

- Distracting or low-trust
- Broken details
- Obvious AI artifacts
- Wrong style or wrong information density for the page

## Replacement Triggers

- Existing asset has dirty edges or bad transparency
- Existing asset clashes with page style
- Existing asset looks too generic or AI-heavy
- Existing asset lowers trust or readability
- Existing asset steals focus from primary content
